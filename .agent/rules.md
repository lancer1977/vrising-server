# Rules for AI Agents

## Working in this Repo

### 1. Naming Tasks
- **Feature Prefixing**: Always use a feature prefix (e.g., `feature-name-task-description.md`) to keep related work grouped together.

### 2. Activity Documentation
- **Activity Log**: Always document significant changes and new features in `.agent/activity.md`. Group entries by date.

### 3. File Hygiene
- **Locks**: Never ignore a `.lock` file. If a card is locked, skip it.
- **Stage Progression**: Always follow the standard stage progression. Never skip stages.

### 3. Workflow Logic
- Read `DEV_STUDIO.md` and `AGENTS.md` in the root for project-specific rules.
